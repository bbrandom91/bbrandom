Probability of Device's Failure¶

Premise: A company has a fleet of devices transmitting daily aggregated telemetry attributes. Predictive maintenance techniques are designed to help determine the condition of in-service equipment in order to predict when maintenance should be performed. This promises cost saving over routine or time-based preventive maintenance should be performed. This approach promises cost savings over routine or time-based preventive mainteance, because tasks are performed only when warranted

1. Overview:¶

  • The purpose of this project is to determine the chance of device's failure ultilizing Machine Learning Technique.
  • In addition, glancing through the set of data, each row of the original data represents a record of the status of a devices which listed in attributes # at the end of the day
  • By determining how each attributes change, we can create a Machine learning model that would predict the probability of devices' failure

def score_result(y, pred):
    print 'Precision Score is', precision_score(y, pred)
    print 'Recall Score is', recall_score(y, pred)
    print 'f1_score is ', f1_score(y, pred)

def f1_score_1(model, fold = 5):
    f1_score = cross_val_score(estimator=model, cv=fold,X=x, y=y,scoring='f1')
    return f1_score
def precision_1(model, fold = 5):
    precision = cross_val_score(estimator=model, cv=fold,X=x, y=y,scoring='precision')
    return precision
def recall_s(model, fold = 5):
    recall_s = cross_val_score(estimator=model, cv=fold,X=x, y=y,scoring='recall')
    return recall_s
def Scoring(model):
    print "f1 score is: ", f1_score_1(model).mean()
    print "precision is: ", precision_1(model).mean()
    print "recall is: ", recall_s(model).mean()
    
def modelfit(alg, dtrain, target, performCV=True, printFeatureImportance=True, cv_folds=5):
    #Fit the algorithm on the data
    alg.fit(dtrain, target)
        
    #Predict training set:
    dtrain_predictions = alg.predict(dtrain)
    dtrain_predprob = alg.predict_proba(dtrain)[:,1]
    
    #Perform cross-validation:
    if performCV:
        cv_score = cross_val_score(alg, dtrain, target, cv=cv_folds, scoring='recall')
    
    #Print model report:
    if performCV:
        print "CV Score : Mean - %.7g | Std - %.7g | Min - %.7g | Max - %.7g" % (np.mean(cv_score),np.std(cv_score),np.min(cv_score),np.max(cv_score))
        
    #Print Feature Importance:
    if printFeatureImportance:
        #feat_imp = pd.Series(alg.feature_importances_, dtrain.columns.tolist()).sort_values(ascending=False)
        #feat_imp.plot(kind='bar', title='Feature Importances')

        feat_imp = pd.DataFrame(alg.feature_importances_, columns=['value'])
        x_columns = pd.DataFrame(dtrain.columns, columns = ['feature'])
        feat_imp = pd.concat([x_columns,feat_imp], axis = 1).sort_values(by = 'value', ascending = False)

So coming back to our business question, was we able to predict which devices have a high chance of failing? Yes. How would this help the business saving cost? Well, when you look at the big picture, imagine we have 1000 devices, and 100 of them will fail in an interval of less than a year. To minimize the chance of them failing, one would have to do monthly or yearly maintanance on 1000 devices. However with our machine learning algorithm, it we help us identified at least 75% of the devices without manually performing 1000 check up. This will definitely save cost, and will keep the devices in the field longer.

SUMMARY:¶

Through this post, I used many Data Science techniques that worth rementioning. For example, We compare 3 models Gradient Boosting, Adaboosting, and XGBoost. In addition, we can't use accuracy as a score since we have imbalanced class. To understand our data better, we recognized that each row of the original data frame is data obtained at the end of each day the devices are in the field. Therefore, we peformed groupby to calculate the rate of change of each attribute. To explained this better, imagine attribute 1 is actually thermometer tracking, if the devices get hotter each day, these devices might have more chance to fail. Furthermore, with this project we also used PCA to combined couple of attributes to improve our score.
